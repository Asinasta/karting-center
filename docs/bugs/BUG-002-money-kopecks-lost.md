# BUG-002. Цена теряла копейки при форматировании `Money`

## Симптом

В карточках заездов и форме бронирования цены отображались без копеек. Например, сумма `150050` коп. (1500,50 ₽) показывалась как «1500 ₽» вместо «1 500,50 ₽».

## Требования

- `01-analysis/api/common/models.yaml` — `Money.amount` в **копейках**.
- LOGIC-003 — цена в RUB; отображение должно соответствовать контракту API.

## Причина

В `client/lib/features/slots/domain/slot_models.dart` метод `Money.formatted` делил `amount / 100` и вызывал `toStringAsFixed(0)`, отбрасывая дробную часть.

```dart
// Было (ошибочно):
final rubles = amount / 100;
return '${rubles.toStringAsFixed(0)} ₽';
```

## Исправление

Логика перенесена в `client/lib/core/domain/common_models.dart`:

- целые рубли и копейки считаются отдельно (`amount ~/ 100`, `amount % 100`);
- копейки выводятся только если ненулевые;
- добавлено разделение тысяч для читаемости.
