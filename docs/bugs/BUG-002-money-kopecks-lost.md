# BUG-002. Цена теряла копейки при форматировании `Money`

## Симптом

В карточках заездов и форме бронирования цены отображались без копеек. Например, сумма `150050` коп. (1500,50 ₽) показывалась как «1500 ₽» вместо «1 500,50 ₽».

## Требования

- `01-analysis/api/common/models.yaml` — `Money.amount` в **копейках**.
- `01-analysis/5-mobile-app-spec/09_Логики/LOGIC-003_Расчёт-цены.md` — цена в RUB; отображение должно соответствовать контракту API.

## Причина

В черновой реализации `Money.formatted` сумма в копейках делилась на 100 с отбрасыванием дробной части (`toStringAsFixed(0)`), из‑за чего ненулевые копейки не отображались.

```dart
// Было (ошибочно):
final rubles = amount / 100;
return '${rubles.toStringAsFixed(0)} ₽';
```

## Исправление

Логика вынесена в `client/lib/core/domain/common_models.dart` и переиспользуется во всех фичах через `Money`:

- целые рубли и копейки считаются отдельно (`amount ~/ 100`, `amount % 100`);
- копейки выводятся только если ненулевые;
- добавлено разделение тысяч для читаемости (`1 500,50 ₽`).

## Проверка вручную

1. Открыть карточку заезда с ценой из fixtures (например 1500 ₽ = 150000 коп.) — отображение «1 500 ₽».
2. Для суммы с копейками (например `150050` коп.) — формат «1 500,50 ₽» в карточке слота, форме бронирования и итоге.
