# Анализ · «Картинг-центр Апекс»

Артефакты аналитика по проекту клиентского мобильного приложения для записи на заезды в картинг-центре «Апекс». 
От сырого брифа заказчика до требований, дизайн-брифа, модели данных, OpenAPI и ТЗ на приложение.

**Канон для разработки:** [`5-mobile-app-spec/`](5-mobile-app-spec/) (экраны, логики, feature-list). Папка [`3-design-brief/`](3-design-brief/) — UX-слой для дизайнера; при расхождениях приоритет у `5-mobile-app-spec/` и реализации в `backend/` + `client/`.

## Скоуп

В текущую поставку входит только клиентское мобильное приложение и API для него. Слоты, конфигурации трасс и маршалы поступают из существующего бэкенда в режиме read-only. Список и карточки слотов доступны без авторизации; бронирование и персональные разделы требуют входа. Интерфейсы маршала, владельца и администратора находятся вне скоупа текущей поставки. Оценки маршалов и программа лояльности входят в MVP по брифу заказчика.

Технология реализации клиента: **Flutter + Dart**. Требования к продукту остаются платформенно-независимыми, но ТЗ на мобильное приложение учитывает Flutter, защищённое хранение токенов, системный push и интеграцию карт через согласованный Flutter-плагин или fallback.

## Важное правило по числам

Числа из брифа фиксируются как факты: 14 картов, новичковый заезд не более 8 участников, расписание на неделю вперёд. Числа, которых нет в брифе (например, порог ранней отмены 2 часа, push-напоминания за 24 и 2 часа), помечены как **предположения** в `1-elicitation/customer-questions.md` и не выдаются за ответы заказчика.
