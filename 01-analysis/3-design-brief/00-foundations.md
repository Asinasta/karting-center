# Foundations

> Сквозные правила дизайна клиентского приложения «Апекс».

## Продукт и аудитория

Клиент пользуется приложением с телефона перед поездкой или прямо у трассы. Контекст может быть шумным, на улице, при ярком солнце, с перчатками или мокрыми руками. Поэтому интерфейс должен быть быстрым, контрастным и не требовать долгого ввода.

## Навигация

Основная авторизованная зона:

- **Запись** — список слотов и фильтры (в раннем UX-дизайне таб мог называться «Заезды»; канон продукта — «Запись», см. `5-mobile-app-spec/`).
- **Мои записи** — история и детали броней.
- **Профиль** — данные клиента и настройки.

Неавторизованная зона: SCR-001 «Регистрация / вход».

## Состояния экранов

Каждый экран должен иметь состояния:

- Loading — загрузка данных.
- Content — нормальное содержимое.
- Empty — данных нет.
- Error — ошибка сети/API с кнопкой «Повторить».
- Offline stale — показан кэш, данные могли устареть.

## Микрокопия

- Empty list: «Пока нет доступных заездов».
- Slot full: «Мест на этот заезд уже нет».
- Rental unavailable: «Прокатной экипировки не хватает».
- Center cancelled: «Заезд отменён центром».
- Offline mutation: «Для записи или отмены нужно подключение к интернету».

## Доступность

- Touch-target не меньше 44pt.
- Статусы не передаются только цветом.
- Поддерживается системный размер шрифта.
- Все интерактивные элементы имеют понятные labels для VoiceOver/TalkBack.

## Карта трассы

На карточке заезда и деталях брони показывается место сбора и схема трассы. Если карта недоступна, отображаются адрес/ориентир и координаты. Для Flutter допускается карта через согласованный Flutter-плагин карт **(предположение: Яндекс.Карты или аналог)**; текстовый fallback обязателен.

## Глоссарий UI

| Термин в интерфейсе | Значение |
| :-- | :-- |
| Заезд | Запланированный старт группы |
| Конфигурация трассы | Вариант трассы для заезда |
| Маршал | Инструктор на трассе |
| Экипировка | Шлем + подшлемник |
| Место сбора | Точка, куда клиент должен прийти перед инструктажем |
