# Design review

> Самопроверка дизайн-брифа до передачи в макеты.

## Проверки

| ID | Проверка | Статус |
| :-- | :-- | :-- |
| RR-D01 | Во всех экранах используется термин «заезд», а не «прогулка» | OK |
| RR-D02 | Маршал не получает отдельный клиентский интерфейс | OK |
| RR-D03 | Лимит мест на бронь = min(free_seats, capacity_cap) | OK |
| RR-D04 | Порог отмены 2 часа помечен как предположение | OK |
| RR-D05 | Карта имеет fallback без интерактивного провайдера | OK |
| RR-D06 | Оплата в MVP только офлайн | OK |
| RR-D07 | Flutter учтён в технических ограничениях, но UX не завязан на конкретную библиотеку | OK |
| RR-D08 | Список и карточки заездов доступны без авторизации, запись требует входа | OK |
| RR-D09 | Смена телефона в профиле вынесена в отдельный OTP-flow | OK |
| RR-D10 | Push об отмене центром отделён от push-напоминаний | OK |

## Открытые вопросы для заказчика

- Подтвердить порог ранней отмены.
- Подтвердить число прокатных комплектов экипировки.
- Подтвердить, показывать ли имя маршала клиентам в списке или только в карточке.
