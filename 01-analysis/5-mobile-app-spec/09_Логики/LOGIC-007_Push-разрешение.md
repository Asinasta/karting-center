# LOGIC-007 — Push-разрешение

## Где используется

BS-002, SCR-007.

## Правила

- Запрос системного разрешения показывается после первой успешной брони.
- При разрешении Flutter получает push token и вызывает `registerPushToken`.
- Если пользователь отказал, приложение работает без push.
- Push об отмене центром — обязательный MVP-сценарий при доступном разрешении.
- Push-напоминания о записи — Should-сценарий; тайминги `[24, 2]` — предположение и должны приходить из API/конфига.
- При нажатии на push об отмене или напоминании приложение открывает детали соответствующей брони.
- При обновлении push token клиент повторно вызывает `registerPushToken`.

## Flutter-заметки

Использовать системный push через согласованный Flutter-плагин. Не блокировать основной сценарий записи, если push недоступен.
