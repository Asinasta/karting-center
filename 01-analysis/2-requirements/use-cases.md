# Use cases / сценарии

> Этап 2. Детальные сценарии взаимодействия клиента с системой. Включают альтернативные потоки и ошибки.

## UC-1: Запись на заезд

- **Актор:** Клиент
- **Предусловия:** Клиент авторизован; есть слот в статусе `scheduled`.
- **Триггер:** Клиент хочет забронировать место на заезд.

### Основной поток

1. Клиент открывает список слотов.
2. При необходимости применяет фильтры.
3. Открывает карточку заезда.
4. Выбирает число мест: 1–3 (лимит 3 — предположение).
5. Для каждого места выбирает экипировку: своя / прокатная.
6. Нажимает «Записаться».
7. Flutter-клиент отправляет `POST /bookings` с `Idempotency-Key`.
8. API атомарно проверяет места и прокатную экипировку.
9. API создаёт бронь, фиксирует `price_total`.
10. Клиент видит экран успеха и бронь в «Моих бронированиях».

### Альтернативные потоки

- **A1. Вся экипировка своя.** `rental_count = 0`, прокатный фонд не уменьшается.
- **A2. Часть мест с прокатом.** Проверяется только нужное число прокатных комплектов.
- **A3. Фильтр не дал результатов.** Показывается empty state и действие «Сбросить фильтры».

### Исключения

- **E1. Не хватает мест.** API возвращает `409 slot_full`, клиент показывает актуальные свободные места.
- **E2. Не хватает прокатной экипировки.** API возвращает `409 slot_full` с `available_rental_gear`, клиент предлагает уменьшить прокат.
- **E3. Слот заполнился параллельно.** API отклоняет запрос, клиент обновляет слот.
- **E4. Слот отменён центром.** API возвращает `410 slot_cancelled`, CTA записи отключается.
- **E5. Таймаут.** Клиент повторяет запрос с тем же `Idempotency-Key`, дубль не создаётся.

### Постусловия

- При успехе создана активная бронь.
- При ошибке состояние слота не изменилось.

## UC-2: Отмена брони

- **Актор:** Клиент
- **Предусловия:** У клиента есть предстоящая активная бронь.
- **Триггер:** Клиент решает отменить участие.

### Основной поток

1. Клиент открывает «Мои бронирования».
2. Выбирает активную бронь.
3. Нажимает «Отменить».
4. Система показывает подтверждение отмены всей брони.
5. Клиент подтверждает.
6. API определяет тип отмены по серверному времени.
7. При ранней отмене бронь переходит в `cancelled`, места и прокатная экипировка возвращаются в слот.
8. Клиент видит обновлённый статус.

### Альтернативные потоки

- **A1. Поздняя отмена.** Если до старта меньше порога ранней отмены, бронь получает `late_cancel`, места не освобождаются.
- **A2. Клиентский прогноз отличается от серверного.** Применяется статус, который вернул сервер.

### Исключения

- **E1. Заезд уже стартовал.** API возвращает `slot_started`; отмена недоступна.
- **E2. Бронь уже отменена.** API возвращает `already_cancelled`; клиент показывает текущий статус.
- **E3. Нет сети.** Отмена офлайн запрещена; показывается Error/Retry.

## UC-3: Просмотр и фильтрация слотов

- **Актор:** Клиент
- **Предусловия:** Клиент авторизован.
- **Триггер:** Клиент хочет найти подходящий заезд.

### Основной поток

1. Клиент открывает экран «Заезды».
2. Приложение запрашивает `GET /slots` с дефолтом: ближайшие 7 дней.
3. Список отображается по возрастанию времени старта.
4. Клиент открывает фильтры.
5. Выбирает дату/период, тип трассы, маршала, «только свободные».
6. Приложение обновляет список.

### Альтернативные потоки

- **A1. Сброс фильтров.** Клиент возвращается к дефолтному списку на 7 дней.
- **A2. Только свободные.** Заполненные и отменённые слоты скрываются.
- **A3. Нет справочника маршалов.** Фильтр по маршалу скрывается.

### Исключения

- **E1. Нет слотов.** Empty state «Пока нет доступных заездов».
- **E2. Сеть недоступна.** Показывается кэш с пометкой «данные могли устареть».
- **E3. Ошибка сервера.** Показывается единый Error/Retry.

## Матрица ошибок ключевых операций

| Код | HTTP | Операция | UX-реакция | Retry |
| :-- | :-- | :-- | :-- | :-- |
| `slot_full` | 409 | createBooking | Показать актуальные места/экипировку | Нет, изменить параметры |
| `double_booking` | 409 | createBooking | Перейти к существующей брони | Нет |
| `slot_cancelled` | 410 | createBooking | Показать, что заезд отменён центром | Нет |
| `slot_started` | 422 | create/cancel | Сообщить, что заезд уже начался | Нет |
| `already_cancelled` | 409 | cancelBooking | Показать текущий статус | Нет |
| `invalid_code` | 400/401 | verifyOtp | Оставить поле кода активным | Да |
| `rate_limit` | 429 | auth | Показать время до повтора | После `retry_after` |
| `unauthorized` | 401 | any | Refresh token или переход на вход | Да |
| `server_error` | 5xx | any | Error/Retry; для брони — тот же Idempotency-Key | Да |
