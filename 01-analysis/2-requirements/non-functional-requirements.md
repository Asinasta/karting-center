## Ограничения реализации

- Сетевой слой должен обрабатывать `401` через refresh token и повтор запроса только для безопасных сценариев.
- Просмотр списка и карточек слотов не требует access token; персональные запросы и мутации требуют Bearer auth.
- Для `createBooking` обязателен `Idempotency-Key`, чтобы повтор после таймаута не создавал дубль.
- Состояния экранов должны быть единообразными: Loading, Content, Empty, Error, Offline stale.
- Локальное кэширование не является источником истины для мест и цены.
