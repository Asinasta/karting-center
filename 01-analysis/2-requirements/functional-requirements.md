# Функциональные требования

> Этап 2. Что система должна делать. Скоуп — клиентское Flutter-приложение и API для него.

Приоритеты: Must — обязательно к запуску; Should — желательно; Could — развитие; Won't — вне MVP.

## Границы скоупа

- Клиент видит слоты, конфигурации трасс и маршалов как read-only данные из существующего API.
- Клиент не создаёт и не редактирует расписание.
- В MVP входит только роль Клиент.
