# Бизнес-требования

> Этап 2. Зачем бизнесу продукт: цели, метрики успеха и ограничения верхнего уровня.

Скоуп — клиентское Flutter-приложение и API для него. Интерфейсы маршала, владельца и администратора обеспечиваются существующей инфраструктурой.

## Бизнес-требования

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| BR-1 | Устранить ручную путаницу и двойные записи, возникающие при ведении записи через Telegram и доску маркером. | Must | Бриф |
| BR-2 | Дать клиентам возможность самостоятельно записываться на заезды онлайн без ручного посредничества владельца. | Must | Бриф |
| BR-5 | Снизить простой картов из-за поздних отмен за счёт понятной политики отмен, фиксации `late_cancel` и передачи поздних отмен в существующую инфраструктуру/админку. Порог 2 часа — предположение для MVP. | Must | Бриф; R-021; R-032 |
| BR-7 | Запустить MVP к старту высокого сезона примерно за 2 месяца при ограниченном бюджете. | Must | Бриф |
| BR-8 | На старте оставить оплату офлайн (наличные / перевод); приложение фиксирует бронь и показывает цену. | Must | Бриф |
| BR-9 | Клиентское приложение должно быть мобильным и работать на iOS/Android; выбранная технология реализации — Flutter. | Must | Бриф; решение команды |
| BR-10 | Снизить барьер входа: клиент может посмотреть расписание и карточки заездов до авторизации, а входит только перед бронированием или просмотром персональных данных. | Must | R-029 |

## Метрики успеха

| ID | Метрика | Целевой ориентир | Связь |
| :-- | :-- | :-- | :-- |
| M-1 | Доля записей, сделанных клиентами самостоятельно онлайн | ≥ 70% к концу первого сезона (предположение) | BR-2 |
| M-2 | Число двойных броней / конфликтов по местам | 0 | BR-1 |
| M-4 | Доля поздних отмен от всех броней | Снижение относительно текущего уровня; целевой ориентир ≤ 10% (предположение) | BR-5 |
| M-6 | Готовность к старту сезона | MVP запущен в срок около 2 месяцев | BR-7 |

## Ограничения верхнего уровня

- **MVP-first.** В срок входят просмотр слотов, запись, отмена, мои бронирования, профиль, push-уведомления.
- **Публичное расписание.** Список и карточки слотов доступны без входа; персональные операции требуют авторизации.
- **Flutter.** Мобильный клиент реализуется на Flutter + Dart для iOS и Android.
- **Оплата офлайн.** Онлайн-оплата — Phase 2.
- **Существующий бэкенд.** Слоты, маршалы, конфигурации трасс и расчёт доступности приходят из API.
- **Вне скоупа.** Админка, интерфейс маршала, создание/редактирование расписания, авто-погода, лояльность, оценки маршалов.
- **Допущения.** Порог отмены 2 часа, push за 24/2 часа и потолок опытного заезда 14 требуют подтверждения заказчиком.
