# Описание домена и ограничения

> Этап 1. Согласованное описание домена после выявления требований.

## Описание домена

**Бизнес.** «Апекс» — уличный картинг-центр на окраине города. Центр проводит групповые заезды по трассе. Один заезд длится 15–20 минут, перед стартом проводится инструктаж. Есть разные конфигурации трассы: короткая и более простая для новичков, длинная для опытных клиентов.

**Процесс.** Расписание формируется на неделю вперёд в существующей инфраструктуре. Для каждого слота задаются дата и время старта, конфигурация трассы, назначенный маршал-инструктор и доступное число картов. Всего у центра 14 картов. На новичковые заезды ставят не более 8 участников; потолок для опытных заездов до 14 участников фиксируется как предположение и должен быть подтверждён заказчиком.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на заезды: публичный просмотр и фильтрация слотов, бронирование одного или нескольких мест после входа, выбор своей или прокатной экипировки, просмотр своих бронирований, отмена записи, оценка маршала после заезда, программа лояльности для постоянных клиентов и push-уведомления. Управление расписанием, работой маршалов и администрированием остаётся в существующей инфраструктуре.

## Ключевые объекты домена

| Объект | Описание |
| :-- | :-- |
| **Заезд / слот** | Конкретный групповой старт: дата/время, конфигурация трассы, маршал, всего и свободно мест, цена |
| **Конфигурация трассы** | Вариант трассы: новичковая короткая или длинная для опытных; определяет потолок мест |
| **Маршал-инструктор** | Сотрудник, который проводит инструктаж и следит за безопасностью на трассе |
| **Карт** | Единица инвентаря для участия в заезде; всего 14 |
| **Экипировка** | Шлем и подшлемник; может быть своей или прокатной |
| **Бронь** | Запись клиента на слот: число мест, прокатная/своя экипировка, статус, итоговая цена |
| **Клиент** | Пользователь мобильного приложения, который записывается на заезды |

## Пользователи и роли

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость картинг-центра, пользуется приложением с телефона | Найти заезд, записаться, выбрать экипировку, отменить запись, получать уведомления | Да |
| Маршал | Ведёт заезд, проводит инструктаж, следит за трассой | Работает через существующий интерфейс | Нет |
| Владелец / администратор | Управляет расписанием, слотами, персоналом и ручными правками | Работает через существующую админку | Нет |

## Ограничения

### Бизнес-ограничения

- Всего **14 картов** — факт из брифа.
- Новичковый заезд — **не более 8 участников** — факт из брифа.
- Опытный заезд — **до 14 участников** — предположение по числу картов.
- Лимит мест слота и лимит прокатной экипировки считаются отдельно: наличие места не означает наличие прокатного комплекта.
- По умолчанию показываются слоты на ближайшие **7 дней** (R-027).
- Список и карточки слотов доступны без авторизации; вход нужен для бронирования и персональных данных (R-029).
- Ранняя отмена за **2 часа** до старта — предположение для MVP; точный порог нужно подтвердить у Дениса.
- Поздняя отмена фиксируется в истории клиента и передаётся в существующую инфраструктуру/админку без штрафов (R-032).
- Оплата на старте офлайн: наличные или перевод. Онлайн-оплата — Phase 2.
- Если заезд отменён центром или по погоде, бронь получает статус «Отменён центром», клиент получает push, повторная запись на этот слот запрещена (R-008).

### Технические / платформенные ограничения

- Клиент — **Flutter-приложение для iOS и Android**.
- API — REST/OpenAPI; существующий бэкенд является источником истины по слотам, местам, цене, статусам и времени.
- Данные времени передаются в UTC, отображаются в локальной зоне клуба.
- Клиентское приложение не создаёт и не редактирует слоты, конфигурации трасс и маршалов.

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Заезд** | Групповой старт на трассе в конкретное время |
| **Слот** | Запланированный заезд в расписании |
| **Конфигурация трассы** | Вариант трассы: короткая/новичковая или длинная/опытная |
| **Маршал** | Инструктор на трассе, отвечающий за инструктаж и безопасность |
| **Прокатная экипировка** | Комплект шлем + подшлемник, выдаваемый центром |
| **Своя экипировка** | Шлем/подшлемник клиента; не занимает прокатный фонд |
| **Ранняя отмена** | Отмена, при которой места возвращаются в слот; порог 2 часа — предположение |
| **Поздняя отмена** | Отмена менее чем за порог ранней отмены; место не освобождается |
| **Отменён центром** | Статус брони, если слот отменён организатором или по погоде |
| **MVP** | Минимальный объём первой поставки: клиентское приложение и API для самостоятельной записи |
| **API** | Контракт REST/OpenAPI между мобильным приложением и существующим бэкендом |
| **OTP** | Одноразовый SMS-код для входа или подтверждения смены телефона |
| **Push** | Системное уведомление мобильного приложения: напоминание или сообщение об отмене центром |
| **Профиль** | Персональный раздел клиента: имя, телефон, push-настройки, выход и удаление аккаунта |
| **Offline stale** | Состояние, когда приложение показывает ранее загруженные данные с предупреждением, что они могли устареть |
| **Idempotency-Key** | Ключ повторяемости запроса создания брони, чтобы повтор после таймаута не создал дубль |
| **Snapshot брони** | Зафиксированные параметры слота в составе брони для корректного отображения истории |
| **Price preview** | Предварительный локальный расчёт цены до создания брони; не является источником истины |
| **Бронь** | Запись клиента на слот: число мест, выбор экипировки, статус, итоговая цена `price_total`, snapshot параметров слота |
| **Оценка маршала** | Оценка 1–5 звёзд и опциональный комментарий клиента по завершённому или активному заезду; хранится в `marshal_rating` брони |
| **Программа лояльности** | Скидка постоянным клиентам по числу завершённых заездов; tier `regular` (≥3) или `vip` (≥8); пороги R-036 — предположение |
| **completed_rides** | Число броней клиента со статусом `completed`; одна бронь = один завершённый заезд; источник — `completed_rides_count` в профиле |
| **Уведомление** | Push-сообщение клиенту: напоминание о записи (`reminder`), отмена центром (`center_cancelled`), напоминание об оценке маршала (`rate_marshal`) |
| **Bearer auth** | Авторизация персональных запросов через `Authorization: Bearer <access_token>` |
